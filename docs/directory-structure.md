# Project Directory Structure

This document summarizes the main directories in `tracking-your-daily-diet` and what each one is responsible for.

## Overview

```text
.
├── amplify/                  # Amplify backend configuration and generated assets
├── docker/                   # Docker definitions for local development
├── e2e/                      # Playwright end-to-end tests
│   ├── tests/                # Test specs
│   └── utils/                # Test utilities/helpers
├── public/                   # Static assets
├── scripts/                  # Development helper scripts
├── src/                      # Main application source code
│   ├── api/                  # API access logic
│   ├── components/           # Shared UI components
│   ├── constants/            # Shared constants
│   ├── features/             # Feature-based modules
│   ├── models/               # Amplify-generated models
│   ├── pages/                # Next.js page routes
│   ├── stores/               # Global state stores
│   ├── styles/               # Global styles
│   ├── ui-components/        # Amplify Studio generated UI
│   └── utils/                # Cross-cutting utilities
├── compose.yml               # Docker Compose configuration
├── package.json              # npm scripts and dependencies
└── README.md                 # Project overview
```

## Details for `src/`

### `src/pages`

- Entry points for Next.js routes.
- Examples: `index.tsx`, `day.tsx`, `week.tsx`, `settings.tsx`, `preset.tsx`.

### `src/features`

- Feature-oriented modules (kebab-case folders) that group UI, logic, types, and helpers together.
- Examples:
  - `daily-goal`
  - `daily-meal-form`
  - `weekly-calories-chart`
  - `preset-meal-form`

### `src/components`

- Reusable shared components used across multiple features.
- Directory names are primarily PascalCase (for example, `Layout`, `LandingPage`, `NutritionSummary`).

### `src/api`

- Domain-specific API operations.
- Examples:
  - `daily-goal/*`
  - `user-meal-preset/*`

### `src/stores`

- Global state management modules shared across pages/features.

### `src/utils`

- Common utility functions (date formatting, number rounding, notifications, and more).

### `src/ui-components` / `src/models`

- Files generated by Amplify.
- Manual edits should be minimized; when possible, regenerate using the Amplify CLI.

## Notes

- Place E2E specs in `e2e/tests` and test helpers in `e2e/utils`.
- Place static assets in `public/`.
- For Docker-based development, refer to `docker/` and `compose.yml`.
